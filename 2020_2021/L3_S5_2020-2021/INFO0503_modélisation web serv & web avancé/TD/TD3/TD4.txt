Application de gestion d'entreposage et de transport de conteneurs de 20 pieds.

--------------------------------------- OBJETS ---------------------------------------
Entrepot :
	- nom : chaîne de caractères
	- hauteur : entier (entre 6 et 8)
	- nb_rangees : entier
	- nb_places_rangees : entier
	- places : TABLEAU de Place
		Place :
		- char_rangee : caractère
		- num_rangee : entier
		- conteneurs : PILE de Conteneur (taille = hauteur entrepôt)
			Conteneur (abstraite)
				- num_id : chaîne de caractères
				- proprietaire_conteneur : Proprietaire
				- poids_total : réel
				- feuille_route : TABLEAU d'Etape
			Conteneur_Simple (hérite de Conteneur)
				- liste_marchandises : TABLEAU de Marchandise
			Conteneur_Refrigere (hérite de Conteneur)
				- liste_marchandises : TABLEAU de Marchandise
				- temperature : réel
				- autonomie : entier (duree_max - date_mise_en_route)
			Conteneur_Liquide (hérite de Conteneur)
				- type_liquide : Type_Liquide (énumération : ALIMENTAIRE, PERISSABLE, TOXIQUE, ...)
				- volume : réel
			Conteneur_Vrac (hérite de Conteneur)
				- type_contenant : Type_Contenant (énumération : GRAIN, POUDRE, VIVANT, ...)
				
				Proprietaire
					- nom_proprietaire : chaîne de caractères
					- adresse_proprietaire : chaîne de caractères
				Etape
					- date_depart : chaîne de caractères (ou Date)
					- lieu_depart : chaîne de caractères
					- date_arrivee : chaîne de caractères (ou Date)
					- lieu_arrivee : chaîne de caractères
				Marchandise
					- id_produit : chaîne de caractères
					- quantite : entier



---------------------------------------- JSON ----------------------------------------
UN FICHIER PAR ENTREPÔT qui est un tableau associatif contenant : 
	- son nom (chaîne de caractères)
	- sa hauteur (entier entre 6 et 8)
	- son nombre de rangées (entier strictement supérieur à 0)
	- son nombre de places par rangée (entier strictement supérieur à 0)
	- un tableau (ses places) de N (N = nombre de places au total) tableaux associatifs (les places) contenant chacun : 
		- sa rangée (un caractère)
		- son numéro dans la rangée (un entier)
		- SI LA PLACE N'EST PAS VIDE, un tableau (sa pile) d'au plus H (H = hauteur de l'entrepôt) tableaux associatifs (les conteneurs) contenant chacun : 
			- son numéro d'identification (chaîne de caractères)
			- son poids total (chaîne de caractères)
			- une tableau associatifs contenant (son propriétaire) : 
				- le nom du propriétaire (chaîne de caractères)
				- l'adresse propriétaire (chaîne de caractères)
			- un tableau (sa feuille de route) de tableaux associatifs (les étapes) contenant chacun : 
					- une date de départ (chaîne de caractères)
					- un lieu de départ (chaîne de caractères)
					- une date d'arrivée (chaîne de caractères)
					- un lieu d'arrivée (chaîne de caractères)
			- UNE DES OPTIONS SUIVANTES :
				OPTION 1 (conteneur simple)
					- un tableau (sa liste de marchandises) de tableaux associatifs (les marchandises) contenant chacun : 
						- son identifiant (chaîne de caractères)
						- sa quantité (entier strictement supérieur à 0)
				OPTION 2 (conteneur réfrigéré)
					- sa température (réel)
					- son autonomie (entier)
					- un tableau (sa liste de marchandises) de tableaux associatifs (les marchandises) contenant chacun : 
						- son identifiant (chaîne de caractères)
						- sa quantité (entier strictement supérieur à 0)
				OPTION 3 (conteneur liquide)
					- son type de liquide (chaîne de caractères)
					- son volume (réel)
				OPTION 4 (conteneur vrac)
					- son type de contenant (chaîne de caractères)
